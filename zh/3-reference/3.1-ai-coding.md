# AI编程支持

## 概述

捷勃特机器人SDK支持通过AI编辑器进行智能编程，让您能够使用自然语言描述来控制机器人、编写代码和调试程序。通过集成 [MCP (Model Context Protocol)](https://github.com/modelcontextprotocol) 服务器，我们为主流AI编辑器提供了完整的SDK编码支持。

## 支持的AI编辑器

我们支持以下AI编辑器：

| AI编辑器 | 支持状态 |
|---------|---------|
| [Cursor](https://cursor.com) | 完全支持 |
| [Windsurf](https://windsurf.com/) | 完全支持 |
| [Trae](https://www.trae.ai) | 完全支持 |
| [Claude Code](https://www.anthropic.com/claude-code) | 完全支持 |

## 快速开始

### 1. 安装必要工具

首先安装 `uv` 包管理器：

::: code-group

```sh [PyPI]
pip install uv
```

```sh [Windows]
irm https://astral.sh/uv/install.ps1 | iex
```

```sh [macOS/Linux]
curl -LsSf https://astral.sh/uv/install.sh | sh
```

:::

如安装出现问题，可参考[官方文档](https://docs.astral.sh/uv/getting-started/installation/#installation-methods)。

::: tip 提示
如果在中国大陆，建议配置 `UV_DEFAULT_INDEX` 环境变量以使用国内镜像源：

```bash
export UV_DEFAULT_INDEX=https://mirrors.tuna.tsinghua.edu.cn/pypi/web/simple
```

:::

### 2. 配置AI编辑器

#### Cursor配置

1. 打开 `Cursor Settings` → `MCP` 选项卡，这将打开 `~/.cursor/mcp.json` 文件
2. 添加以下配置： {#mcp-config}

```json
{
  "mcpServers": {
    "agilebot-sdk-docs": {
      "command": "uvx",
      "args": [
        "--from",
        "mcpdoc",
        "mcpdoc",
        "--urls",
        "AgilebotRobotSDK:https://agilebot-sdk-docs.readthedocs.io/llms.txt",
        "--transport",
        "stdio"
      ]
    }
  }
}
```

3. 在 `Cursor Settings/Rules` 中添加用户规则： {#user-rule}

```
for ANY questions about Agilebot SDK, Agilebot Robot SDK, or 捷勃特机器人SDK, use the agilebot-sdk-docs tool to help answer --
+ call list_doc_sources tool to get the available llms.txt file
+ call fetch_docs tool to read it
+ reflect on the urls in llms.txt
+ reflect on the input question
+ call fetch_docs on any urls relevant to the question
+ use this to answer the question
```

#### Windsurf配置

1. 使用 `Ctrl+L` 打开Cascade
2. 点击 `Configure MCP` 打开配置文件 `~/.codeium/windsurf/mcp_config.json`
3. 添加相同的[配置内容](#mcp-config)
4. 在 `Windsurf Rules/Global rules` 中添加相同的[规则](#user-rule)

#### Trae配置

1. 在 AI 对话窗口右上角点击设置图标 → MCP
2. 点击`+ 添加`按钮的下拉框，选择`手动添加`
3. 添加相同的[配置内容](#mcp-config)
4. 点击设置图标 → 规则，在`个人规则`中添加相同的[规则](#user-rule)
5. 在智能体设置中选择 `Builder with MCP`

::: tip 模型建议
建议使用Claude 3.7及以上版本的模型以获得更好的SDK编程体验和代码生成质量。
:::

## 使用示例

配置完成后，您可以在AI编辑器的对话框中输入自然语言描述来生成代码。以下是一些示例：

### 简单任务示例

在AI编辑器对话框中输入：

```
请使用捷勃特机器人SDK编写Python代码，实现连接机器人并读取R[1]寄存器的值
```

AI编辑器会自动生成包含连接、读取寄存器、错误处理和断开连接的完整代码。

### 复杂任务示例

在AI编辑器对话框中输入：

```
请使用捷勃特机器人SDK开发一个自动抓取放置系统，包含以下功能：
1. 机器人连接和初始化
2. 移动到拾取位置并抓取物料
3. 移动到放置位置并释放物料
4. 状态监控和数据记录
5. 完整的错误处理机制
```

AI编辑器会生成一个综合运用多种SDK功能的程序，包括：

* **运动控制** - 多点位运动规划和执行
* **IO操作** - 夹具控制和传感器反馈
* **状态监控** - 机器人状态、伺服状态、报警检查
* **数据记录** - 任务计数和时间戳记录到寄存器
* **错误处理** - 每个步骤的状态检查和异常处理
* **安全机制** - 系统初始化和安全关闭流程
