# 关于插件

## 插件是什么

* 插件系统提供了一个完整的框架，可以让生态合作方、外部开发者按照一定的规则开发应用并运行在捷勃特的机器人中。
* Copper/Bronze系统允许运行3种用户自定义的程序： Web小程序、简单服务、通用服务。

## 插件分类

### Web小程序 `webMiniProgram`

Web小程序允许开发者通过Web技术（如HTML、CSS、JavaScript）编写插件的前端界面。并将其嵌入到系统的 Web 页面中，作为前端页面进行展示和交互。

### 简单服务 `easyService`

简单服务插件是一个通过编写一个或多个 Python 方法来实现功能的插件。它允许开发者快速实现一个简单的后端功能，通过 HTTP 请求与系统交互。简单服务通常不涉及复杂的框架或外部依赖，只需要编写一些方法即可。

### 通用服务 `generalService`

通用服务插件可提供更强大的功能支持。它通常是一个功能更复杂的后端服务，能够进行数据处理、复杂的逻辑运算，或者与外部系统交互。

## 插件文件组成

### 配置文件

任何类型的插件必须包含 `config.json` 文件，即[配置文件](/03-reference/01-config)。只有包含配置文件才能安装、初始化、运行。

### 数据文件

后台插件包中一般都包含一个data目录，其中用于存储插件运行时生成的数据。具体可参考[数据持久化](/02-development/03-advanced/04-data-persistence)。
